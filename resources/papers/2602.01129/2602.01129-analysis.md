## Paper Summary

This paper proposes **SMCP (Secure Model Context Protocol)**, a comprehensive security extension to MCP adding unified identity management, robust mutual authentication, ongoing security context propagation, fine-grained policy enforcement, and comprehensive audit logging.

**Core contribution:** Protocol-level security framework addressing systemic challenges rather than point solutions.

**Key insight:** MCP security requires a holistic approach - unified trust, fragmented authentication, and limited auditability are root causes of vulnerabilities.

## Defense/Mitigation Proposals

### SMCP Architecture (5 Core Components)

#### 1. Trusted Component Registry

**Unified Digital Identity Infrastructure:**
- All entities (users, agents, tools, resources) assigned 32-character identity codes
- Character set: [0-9A-Z]
- MOD 97-10 checksum for validation

**Identity Code Structure:**
```json
{
  "identityCode": "AG01AC01COMP2500012345005678X7A3",
  "type": "AG",           // AG=Agent, US=User, TL=Tool, MD=Model
  "version": "01",
  "authority": "AC01",
  "registrant": "COMP",
  "year": "25",
  "packageSerial": "00012345",
  "instanceSerial": "005678",
  "checksum": "X7A3"
}
```

**Registry Types:**
| Type | Example | Identity/Account | Credential |
|------|---------|------------------|------------|
| User | Human, org account | Profile, roles | Login secret, certificate |
| Agent | General/vertical agent | Code hash, boundaries | Public key, capabilities |
| MCP Server | Tool services | Provider, sensitivity | Service certificate |
| Resource | Model API, dataset | Owner, provenance | Resource-level credential |

#### 2. Session Establishment and Mutual Authentication

**Authentication Flow:**
1. Subject initiates request
2. Relying party evaluates access control policy
3. Requester assembles credential bundle, signs challenges
4. Verifier checks signatures, trust chains, validity, revocation
5. Authentication assertion generated
6. Session established with security context binding

**Mutual TLS or application-level assertions supported**

#### 3. Security Context Propagation

**Context Fields:**
| Field | Example Name | Description |
|-------|--------------|-------------|
| Session ID | sessionId | Session identifier |
| Call Chain ID | callChainId | Cross-boundary tracing |
| Delegation Chain | delegatorChain | Authorization provenance |
| Caller Identity | callerAgentId | Requesting agent |
| Peer Identity | peerId | Target identity |
| Authentication Assertion | authnAssertionId | Auth event reference |
| Risk Level | riskLevel | Dynamic risk assessment |
| Policy Reference | policyRef | Applicable policies |
| Data Sensitivity | dataSensitivity | Classification level |

**Key feature:** Context explicitly carried or traceable from sessions, tasks, messages, tool invocations.

#### 4. Fine-Grained Policy Enforcement

- Minimal privilege support
- Flexible protection
- Dynamic policy evaluation
- Per-operation authorization

#### 5. Comprehensive Audit Logging

- All important actions traceable
- Start-to-finish accountability
- Forensic review capability
- Compliance support

### Identity Lifecycle Management

1. **Registration** - Sponsor submits request with functional descriptions, environment digests, behavioral boundaries
2. **Verification** - Risk evaluation, evidence verification
3. **Credential Issuance** - Digital credentials generated (certificates or verifiable credentials)
4. **Activation** - Identity code and credential returned, entity becomes trusted participant
5. **Update** - Identity code preserved, info updated, credentials rotated
6. **Lock/Revocation** - For misbehavior, leaks, delegator revocation

### SMCP Security Context Structure

- Session identifier
- Call chain identifier
- Delegation chain
- Risk level

### Identity Code Encoding Rules

- 32 characters fixed length
- Character set: [0-9A-Z]
- Type field distinguishes Users, Agents, Tools, Models, Infrastructure
- MOD 97-10 checksum
- Cross-type, parseable
- Global uniqueness

### Four Core Roles in SMCP

1. **User** - Human or organizational account
2. **Agent (MCP Host)** - LLM-based orchestrator
3. **MCP Server** - Tool/resource provider
4. **Resource** - Data source or external service
